# 소개팅 안전 점검 웹앱 설계안 (가칭: "미리, 안심")

> 대상: 소개팅 전 상대 정보를 사전에 점검하고 싶은 30대 여성
> 목표: **폭로(exposure)가 아닌 위험 신호 탐지(risk-signal detection)**
> 핵심 제약: 실명 공개 / 공개 검색(외부 신상 조회) / 피해자 게시판 **전면 제외**

---

## 1. 서비스 컨셉 & 포지셔닝

- **한 줄 정의**: "상대를 비난하거나 신상을 캐는 도구가 아니라, 내가 받은 정보(대화·번호·링크·계좌·사진)를 내가 분석해 스스로를 보호하는 셀프 점검 도구"
- **핵심 메시지**: "이 사람이 누구인지가 아니라, 이 사람이 보내는 신호가 위험한지"를 본다.
- **분석 결과의 성격**:
  - 단정적 표현("이 사람은 사기꾼") 금지
  - "이런 패턴은 일반적으로 ○○한 경우에 자주 나타납니다" 식의 **확률적·참고적 안내**
  - 결과는 본인만 열람 (1:1, 비공개)
- **차별점**: 더치트류 서비스처럼 "신고자 글 + 실명/계좌 공개 검색"이 아니라, **입력된 데이터 자체의 패턴/형식/행동 신호**를 분석하는 구조 → 명예훼손·정보통신망법 리스크를 구조적으로 차단

---

## 2. 법적 리스크 관리 원칙 — 제외 기능 vs 대체 설계

| 입력 항목 | ❌ 제외 (법적 리스크) | ✅ 대체 설계 (위험 신호 탐지) |
|---|---|---|
| 대화 내용 | 상대 SNS/실명 역추적 | 러브바밍 속도, 금전·계좌 언급 시점, 동일 스크립트 반복(타사 피해자 사례와 유사한 멘트), 신상정보 진술 불일치 탐지 |
| 전화번호 | 신고 DB 실명 조회, 공개 게시판 검색 | 번호 **형식/유형** 기반 분석(인터넷전화 070, 안심번호 050, 해외번호, 가상번호 패턴) + "번호를 자주 바꾼다"는 사용자 진술 기반 행동 신호 |
| 링크 | (해당 없음 — 링크는 인물이 아닌 기술 자원) | URL 피싱·악성코드 검사, 단축 URL 원본 추적, 도메인 생성일·등록 정보 등 **기술적 위험도** — 가장 법적으로 안전한 카테고리 |
| 계좌번호 | 예금주 실명/신고내역 공개 조회 | 계좌 **형식 유효성** 검증 + "만남 전 계좌 정보를 요구한다" 자체의 **행동 패턴 위험 스코어** (옵션: 익명화된 집계 위험지수 — 신고자/대상자 식별 정보 없이 "최근 N건 유사 패턴 신고" 수준) |
| 사진 | 역방향 이미지 검색으로 실제 인물 신상 노출(무고한 제3자 피해 가능성) | AI 생성/합성(딥페이크) 탐지, 메타데이터·압축 흔적 분석(스크린샷/재다운로드 여부), 내부 DB 내 **익명 해시 중복 등록 카운트**("동일 이미지가 여러 프로필에서 사용된 패턴 N건 감지") |

**공통 운영 원칙**
- 본인이 직접 받은 정보만 입력하도록 안내 ("내가 받은 메시지를 내가 분석")
- 분석 결과 내 어디에도 상대방을 특정할 수 있는 실명·계정명·직장명 노출 없음
- 데이터는 분석 후 자동 삭제(기본값) — 보관은 사용자가 명시적으로 선택할 때만
- 약관/면책 문구에 "범죄사실 단정 아님, 참고 정보" 명시
- 서비스 출시 전 변호사 자문 권장 (개인정보보호법, 정보통신망법, 명예훼손 관련)

## 3. 전체 화면 흐름

```
첫 화면(랜딩)
   └─▶ 동의 화면 (개인정보 처리방침 / 보관기간 선택)
         └─▶ 입력 흐름 (Step 1~5, 전부 선택형)
               └─▶ 분석 중 로딩
                     └─▶ 결과 화면 (무료 요약)
                           ├─▶ 결제 (상세 리포트 잠금 해제)
                           │     └─▶ 결과 화면 (상세, 유료)
                           │            └─▶ PDF/이미지 저장, 히스토리 저장
                           └─▶ (결제 안 함) 종합 등급만 확인 후 종료
```

---

## 4. 첫 화면 (랜딩 / 온보딩)

**상단 (Hero)**
- 메인 카피: "소개팅 가기 전, 5분만 투자하세요"
- 서브 카피: "폭로가 아닌, 나를 위한 사전 점검 — 결과는 나만 봅니다"
- CTA 버튼: `무료로 점검 시작하기`
- 신뢰 배지 3종: `비공개 분석` `자동 삭제` `실명 미노출`

**중단 (점검 항목 소개)**
- 5개 아이콘 카드: 대화 / 전화번호 / 링크 / 계좌 / 사진
- 각 카드에 한 줄 설명 (예: "대화 — 대화 속 위험 신호 패턴 분석")

**작동 방식 (3단계 설명)**
1. 받은 정보 입력 (대화·번호·링크·계좌·사진 중 원하는 것만)
2. AI가 패턴 분석 (러브바밍, 스크립트, 위조 가능성 등)
3. 나만 보는 위험도 리포트 확인

**신뢰/안전 섹션**
- "이 서비스는 특정 개인을 공개하거나 비방하지 않습니다"
- "분석 결과는 통계적 패턴에 기반한 참고 정보입니다"
- FAQ 미리보기 (데이터는 어떻게 처리되나요? / 상대방이 알 수 있나요? 등)

**하단**
- 개인정보 처리방침 / 이용약관 링크
- 재방문 사용자를 위한 `로그인` 버튼 (히스토리 확인용, 선택)

---

## 5. 입력 흐름 (전 단계 스킵 가능한 위저드)

### Step 0. 동의 화면
- 개인정보 처리방침 동의 (필수)
- "입력한 정보는 본인이 직접 수신한 정보임을 확인합니다" 체크박스
- 데이터 보관 옵션 선택: `분석 후 즉시 삭제(기본)` / `24시간 보관` / `계정에 보관(로그인 시)`

### Step 1. 대화 내용 입력 (권장)
- 텍스트 붙여넣기 또는 캡처 이미지 업로드(OCR)
- 분석 포인트 안내: 호감 표현 속도, 금전/계좌 언급 시점, 반복되는 정형 문구, 신상 진술 불일치

### Step 2. 전화번호 입력 (선택)
- 입력 형식: 국가코드 포함 번호
- 분석 포인트 안내: 번호 유형(휴대폰/인터넷전화/안심번호/해외번호), 형식 기반 위험 신호

### Step 3. 링크 입력 (선택)
- 대화 중 공유받은 URL 입력 (복수 가능)
- 분석 포인트 안내: 피싱/악성 사이트 여부, 단축 URL 원본 추적, 도메인 생성일

### Step 4. 계좌번호 입력 (선택)
- 은행명 + 계좌번호 입력
- 분석 포인트 안내: 형식 유효성, "만남 전 계좌를 요구했다"는 행동 자체의 위험도

### Step 5. 사진 업로드 (선택)
- 프로필/대화 중 받은 사진 업로드 (최대 N장)
- 분석 포인트 안내: AI 합성/딥페이크 가능성, 메타데이터 이상 여부, 내부 DB 중복 등록 패턴

**공통 UI**
- 진행률 바 (Step X / 5)
- 각 단계 상단에 "왜 필요한가요?" 툴팁
- "입력한 항목이 많을수록 분석 정확도가 올라갑니다" 안내 배너
- 마지막 단계 후 `분석 시작하기` CTA → 로딩 화면(예상 소요 시간 표시)

## 6. 분석 결과 화면

### 상단: 종합 위험도
- 게이지/등급 형태 (예: `안전` → `관찰 필요` → `주의` → `위험` 4단계, 또는 0~100 스코어)
- 한 줄 요약: "총 N개 카테고리 중 M개에서 위험 신호가 발견되었습니다"

### 카테고리별 카드 (대화 / 전화번호 / 링크 / 계좌 / 사진)
- 각 카드: 위험 신호 건수 배지 + 한 줄 요약
- **무료(Free)**: 종합 등급 + 카드별 "위험 신호 N건 발견" 까지만 노출, 상세 내용은 블러 처리 + 잠금 아이콘
- **유료(Paid)**: 카드 펼치면 —
  - 감지된 패턴 설명 (예: "대화 시작 후 3일 만에 '운명' 표현 4회 — 일반적인 러브바밍 패턴과 유사")
  - 왜 위험할 수 있는지 일반화된 설명
  - 권장 행동 가이드 (예: "직접 만나기 전 영상통화를 요청해보세요", "계좌 정보 요구에는 응하지 마세요")

### 하단
- "지금 할 수 있는 것" 체크리스트 (행동 가이드 모음)
- 결과 PDF/이미지 저장 (유료)
- 면책 문구: "본 분석은 통계적 패턴에 기반한 참고 정보이며, 특정인의 신원이나 범죄 사실을 단정하지 않습니다"

## 7. 결제 전환 구조

### 퍼널
1. **무료 단계**: 종합 등급 + 카테고리별 "위험 신호 N건" 숫자까지 노출 (궁금증 유발) → 상세는 블러+잠금
2. **전환 트리거**: 블러 카드 위에 `상세 분석 결과 보기` CTA, 구체적 숫자("3건의 주요 신호")로 클릭 유도
3. **결제 전 신뢰 요소**:
   - 샘플 리포트 미리보기 (다른 가상 사례)
   - 환불 정책 안내
   - 보안/프라이버시 배지 재노출 ("결제 후에도 신상정보는 저장되지 않습니다")

### 가격 구조 (예시)
- **단건 리포트**: 2,900~4,900원 — 1회 분석 상세 결과 잠금 해제
- **5회 크레딧 패키지**: 9,900원 — 소개팅을 자주 하는 사용자 대상, 1회당 단가 할인
- **월 구독**: 4,900원/월 — 무제한 분석 + 결과 히스토리 보관

### 결제 수단
- 카카오페이, 토스페이, 신용/체크카드 (간편결제 우선 노출)

### 결제 후
- 즉시 잠금 해제 → 상세 리포트 화면으로 전환
- 로그인 사용자: 히스토리에 결과 저장 (보관 옵션 선택 시)
- 업셀: 분석 완료 후 "다음 소개팅도 확인해보세요" 알림, 잔여 크레딧 표시로 재구매 유도

## 8. MVP 우선순위 제안 (참고)

1. **Phase 1** (가장 빠르고 법적으로 안전): 대화 패턴 분석 + 링크 안전성 검사
2. **Phase 2**: 사진 AI 생성 탐지 + 전화번호 형식 기반 분석
3. **Phase 3**: 계좌 행동 패턴 스코어 + 결제/구독 시스템 고도화, 결과 히스토리

---

## 9. 데이터/프라이버시 정책 요약

- 입력 데이터는 분석 목적 외 사용 금지, 기본값은 분석 후 즉시 삭제
- 외부 공개 DB/검색 엔진과 연동한 "신원 조회" 기능 없음
- 결과 화면·리포트 내 상대방 식별 정보(실명, SNS 계정, 직장 등) 노출 없음
- 사용자 간 정보 공유/게시판 기능 없음 (1:1 비공개 결과)
- 서비스 약관에 "참고 정보 제공" 성격 명시, 법률 자문 권장
